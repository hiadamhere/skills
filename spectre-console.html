🚦 The one rule that prevents hangs

Spectre's interactive features — every prompt and the Status/Progress/Live displays — assume a real terminal. Under redirected stdin, in CI, or in a hosted (non-tty) process they throw or block forever waiting for input that never comes. Always gate on capability first, with a non-interactive fallback.

C#
using Spectre.Console;

// Interactive == a real, attached terminal that can read keystrokes.
if (AnsiConsole.Profile.Capabilities.Interactive && AnsiConsole.Profile.Out.IsTerminal)
{
    var name = AnsiConsole.Ask<string>("Your [green]name[/]?");
    AnsiConsole.MarkupLine($"Hello, [bold]{Markup.Escape(name)}[/]!");
}
else
{
    // Non-interactive fallback: plain output, defaults, or read from args/env.
    AnsiConsole.WriteLine("Running non-interactively; using defaults.");
}

AnsiConsole.Profile.Capabilities.Interactive is false when input is redirected; AnsiConsole.Profile.Out.IsTerminal is false when stdout is piped/captured.

📦Output widgets

Everything printed is an IRenderable written with AnsiConsole.Write(renderable); widgets nest inside one another. Columns and rows are fluent extension methods.

C#
using Spectre.Console;

var table = new Table();
table.Border = TableBorder.Rounded;          // Rounded | Square | Minimal | Ascii | Markdown | Heavy …
table.Title = new TableTitle("[yellow]Users[/]");

table.AddColumn("Name");
table.AddColumn(new TableColumn("Age").Centered());
table.AddRow("Alice", "30");
table.AddRow("Bob", "25");

AnsiConsole.Write(table);

// Compose freely: Panel, Tree, Rule, Columns, Rows, Padder, Align, BarChart …
var tree = new Tree("[bold]solution/[/]");
var src = tree.AddNode("[blue]src/[/]");
src.AddNode("Program.cs");
AnsiConsole.Write(new Panel(tree) { Header = new PanelHeader("Files") });

Covered in the skill: Table, Grid, Panel, Tree, Rule, Columns, Rows, Padder, Align, TextPath, FigletText, and charts.

🎨Markup & color — escape untrusted text

Spectre renders styled text from an inline [style]…[/] markup string. Any [ or ] in interpolated or user-supplied text is a control character and will throw or corrupt output — escape it.

C#
using Spectre.Console;

AnsiConsole.MarkupLine("[bold red]Error:[/] disk full");

string userInput = "array[0] = [redacted]";
AnsiConsole.MarkupLine($"You typed: [grey]{Markup.Escape(userInput)}[/]");   // safe

// Or the interpolated helpers, which escape the holes for you:
AnsiConsole.MarkupLineInterpolated($"You typed: [grey]{userInput}[/]");

Markup.Escape doubles brackets; MarkupLineInterpolated / Markup.FromInterpolated escape interpolation holes automatically — the safest option for dynamic text. Color, Style, and Decoration are value types for building styles without hand-written markup.

⏳Live displays — Status, Progress, Live

Three widgets animate a console region. A live display owns the region exclusively — never write to the console directly or start a second live display from inside the callback; mutate through the supplied context instead.

C#
using Spectre.Console;

AnsiConsole.Progress()
    .AutoClear(false)
    .Columns(new ProgressColumn[]
    {
        new TaskDescriptionColumn(),
        new ProgressBarColumn(),
        new PercentageColumn(),
        new SpinnerColumn(),
    })
    .Start(ctx =>
    {
        var download = ctx.AddTask("[green]Downloading[/]");   // MaxValue defaults to 100
        while (!ctx.IsFinished)
        {
            download.Increment(2.5);
            // …pace the loop / await real work…
        }
    });

Status() shows a spinner + message; Live(renderable) re-renders any widget in place via ctx.Refresh(). Async variants (StartAsync) take a Task-returning callback.

⌨️Prompts

Four interactive prompts: free-text (TextPrompt<T>), yes/no (ConfirmationPrompt), single-choice (SelectionPrompt<T>), and multi-choice (MultiSelectionPrompt<T>) — run via AnsiConsole.Prompt(...) or the Ask/Confirm shortcuts. All require a real terminal (gate first).

C#
using Spectre.Console;

string env = AnsiConsole.Prompt(
    new SelectionPrompt<string>()
        .Title("Choose an [green]environment[/]")
        .PageSize(10)
        .EnableSearch()
        .AddChoices("dev", "staging", "production"));

int age = AnsiConsole.Ask<int>("Your [green]age[/]?");
bool ok = AnsiConsole.Confirm("Deploy to [red]production[/]?", defaultValue: false);

TextPrompt<T> supports .Validate(...), .Secret(), .DefaultValue(...), and .AllowEmpty(); MultiSelectionPrompt<T> returns a List<T> with .Required() / .NotRequired().

🧪Layout, DI & testing

Prefer an injected IAnsiConsole over the static AnsiConsole facade — it is the seam that makes output testable. The Spectre.Console.Testing package's TestConsole captures output and can feed scripted input.

C#
using Spectre.Console;
using Spectre.Console.Testing;

// Mark the test console interactive so gated prompt code runs.
var console = new TestConsole().Interactive();
console.Input.PushTextWithEnter("Alice");        // scripted keystrokes

string name = console.Prompt(new TextPrompt<string>("Name?"));
// name == "Alice"; inspect rendered output via console.Output / console.Lines

Static output (Table, Panel, MarkupLine) is safe everywhere — it renders without color/animation when unsupported. Only prompts and live displays need gating.
